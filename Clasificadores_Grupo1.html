Luego definimos transformaciones para numricas/categricas:

Para mejorar la calidad del modelo y asegurar que todos los algoritmos puedan interpretar correctamente las variables, se disearon pipelines de preprocesamiento diferenciados segn el tipo de dato:

  • Variables numricas:

    • Se aplic imputacin de valores faltantes usando la mediana, ya que es una medida robusta frente a valores extremos.

    • Posteriormente, se realiz escalado estandarizado (StandardScaler) para que todas las variables numricas quedaran en la misma escala (media = 0, desviacin estndar = 1).

    • Esto es importante porque modelos como SVM y KNN dependen de distancias, y sin normalizacin podran verse sesgados por atributos con magnitudes mayores (por ejemplo, Age vs Fare).

  • Variables categricas:

    • Se imputaron valores faltantes con la moda (valor ms frecuente).

    • Luego se codificaron mediante One-Hot Encoding, creando variables binarias (0/1) por cada categora posible.

    • Esta tcnica evita la interpretacin errnea de relaciones ordinales entre categoras (por ejemplo, entre male y female), y es compatible con modelos lineales o basados en rboles.

Finalmente, ambos pipelines (numrico y categrico) se integraron mediante un ColumnTransformer, lo que permiti automatizar el preprocesamiento dentro del flujo de entrenamiento y validacin de cada modelo. Esto asegura coherencia y reproducibilidad: las mismas transformaciones se aplican tanto en los datos de entrenamiento como en los de prueba.

In[]:
num_features = ["Age", "Fare", "FamilySize"]
cat_features = ["Pclass", "Sex", "Embarked", "CabinLetter"]

num_transformer = Pipeline([
    ("imputer", SimpleImputer(strategy="median")),
    ("scaler", StandardScaler())
])

cat_transformer = Pipeline([
    ("imputer", SimpleImputer(strategy="most_frequent")),
    ("onehot", OneHotEncoder(handle_unknown="ignore"))
])

preprocessor = ColumnTransformer(
    transformers=[
        ("num", num_transformer, num_features),
        ("cat", cat_transformer, cat_features)
    ])
